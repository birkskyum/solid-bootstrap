# Checks and radios

<div class="lead">
  Create consistent cross-browser and cross-device checkboxes and radios with
  our completely rewritten checks component.
</div>

For the non-textual checkbox and radio controls, `FormCheck` provides a single component for both types that adds some additional styling and improved layout.

<div class="row gy-5 pt-3">

  <Example title="Default (stacked)">
    <Example.Intro>
      By default, any number of checkboxes and radios that are immediate sibling will be vertically stacked and appropriately spaced with FormCheck.
    </Example.Intro>
    <Example.Preview>
      <Form>
        {['checkbox', 'radio'].map((type) => (
          <div class="mb-3">
            <Form.Check 
              type={type}
              id={`default-${type}`}
              label={`default ${type}`}
            />

            <Form.Check
              disabled
              type={type}
              label={`disabled ${type}`}
              id={`disabled-default-${type}`}
            />
          </div>

  <Example title="Customizing FormCheck rendering">
    <Example.Intro>
      When you need tighter control, or want to customize how the `FormCheck` component
      renders, it may better to use its constituent parts directly.

      By provided `children` to the `FormCheck` you can forgo the default rendering and
      handle it yourself. (You can still provide an `id` to the `FormCheck` or
      `FormGroup` and have it propagate to the label and input).
    </Example.Intro>
    <Example.Preview>
      <Form>
        {['checkbox', 'radio'].map((type) => (
          <div class="mb-3">
            <Form.Check type={type} id={`check-api-${type}`}>
              <Form.Check.Input type={type} isValid />
              <Form.Check.Label>{`Custom api ${type}`}</Form.Check.Label>
              <Form.Control.Feedback type="valid">You did it!</Form.Control.Feedback>
            </Form.Check>
          </div>
        ))}
      </Form>
